Specialty

Database Engineering

Data layer work is where I invest extra care. Getting the schema right, the queries fast, and the business logic cleanly expressed through the data model is what separates a system that scales from one that becomes a production incident.

I work primarily in SQL Server, with deep experience in schema design, query optimization, indexing strategy, and translating complex business rules into data models that stay maintainable as requirements change. Entity Framework Core is my primary ORM — I know when to lean on it and when to write raw SQL instead.

One of the more demanding projects I've worked on was a closing disclosure viewer built for a major mortgage lender. The tool integrated over 3,000 discrete data fields across a tightly regulated document type, with strict conditional logic governing what could appear under which circumstances. That kind of work demands precise data modeling, rigorous validation at the data layer, and clean APIs that shield the frontend from that complexity entirely.

Schema Design & Data Modeling

Designing normalized schemas that reflect real business domains, with an eye toward future query patterns and evolution.

Query Optimization

Profiling slow queries, reading execution plans, and applying indexing strategies that actually move the needle.

Entity Framework Core

Fluent API configuration, migration management, and knowing when to reach for Dapper or raw SQL for performance-critical paths.

Complex Business Logic at the Data Layer

Encapsulating intricate conditional rules so the application tier stays clean — demonstrated at scale with 3,000+ field integrations for a production mortgage platform.
